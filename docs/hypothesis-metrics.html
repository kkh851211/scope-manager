검증 전략 배경

왜 고객 인터뷰 대신 MVP 즉시 배포를 선택했는가

⚡ 핵심 전략 명제

AI 개발 도구의 발달로 MVP 제작 비용 < 인터뷰 설계/진행 비용의 역전 현상이 발생했다. 말(intention)보다 행동(behavior)이 더 정직한 신호다. 따라서 "만들어서 보여주는 것"이 "물어보는 것"보다 더 빠르고 정확한 검증 방법이다.

❌ 기존 방식 — 인터뷰 우선
말과 행동의 불일치
"쓸 것 같다"는 인터뷰 응답이 실제 사용 행동으로 이어지지 않는 경우가 다반사. False Positive 위험이 높다.
시간 & 비용 소모
인터뷰 대상자 섭외 → 일정 조율 → 진행 → 분석까지 수 주 소요. AI 시대에 이 비용이 MVP 개발보다 커진 역전 현상 발생.
컨텍스트 없는 상상
실물을 보지 못한 상태의 인터뷰이는 자신의 실제 니즈를 정확히 언어화하기 어렵다. 프로토타입이 있어야 진짜 반응이 나온다.
✅ 선택 방식 — MVP 즉시 배포
행동 데이터의 정직함
실제 가입 여부, 첫 피치덱 생성 완료 여부, 재방문 여부. 행동은 거짓말하지 않는다. 이탈 지점이 곧 문제의 좌표다.
비용 역전 현상 활용
AI 도구 덕분에 MVP 개발 비용이 수백만 원 → 수십만 원 이하로 압축. 인터뷰 세팅 비용과 역전.
실물 기반 피드백 루프
배포 후 실사용자의 행동 패턴 → 인터뷰 시나리오보다 훨씬 구체적인 개선 가설 도출 가능.
전략 선택의 전제 조건 (이 전략이 유효한 이유)
🤖
AI 개발 비용 극적 하락
단일 기능(소급 텍스트 입력 + 피치덱 생성)으로 좁혀, AI 도구만으로 수 주 내 배포 가능한 수준의 MVP 설계 완료.
🎯
명확한 얼리어답터 채널
블라인드·단톡방 등 타겟 실무자 집결지가 명확해, 배포 후 초기 트래픽 확보 경로가 존재함.
📏
측정 가능한 이진(Binary) 신호
"피치덱을 완성했는가/안 했는가"처럼 Yes/No로 끊리는 행동 지표가 존재해, 해석의 모호함 없이 가설 검증 가능.

핵심 가설 5선

린 캔버스 각 블록에서 도출된 검증 우선순위 가설 · 위험도 순 정렬

H1
문제 가설 — 실무자의 '증명 불가' 페인은 실재하며, 행동으로 이어질 만큼 강렬하다
위험도 HIGH
가설 내용
KPI에 잡히지 않는 기여를 증명하고 싶은 욕구가 실제 가입 및 첫 입력 시도 행동으로 전환될 만큼 강하다.
검증 방법
블라인드/단톡방 배포 후 랜딩 → 가입 전환율(CVR) 측정. "이 문제 있는 사람?" 설문 반응도 병행.
성공 기준
랜딩→가입 CVR ≥ 15%
첫 이슈 입력 시도율 ≥ 60% (가입자 중)
H2
핵심 기능 가설 — AI가 텍스트를 금액으로 변환한 결과가 "쓸 만하다"고 느껴진다
위험도 HIGH
가설 내용
자유 텍스트 3건을 입력했을 때 AI가 산출한 구조화 결과와 예상 방어 금액이 실무자가 납득할 수 있는 수준으로 출력된다.
검증 방법
AI 산출값을 수정 없이 그대로 제출한 비율 vs. 금액 수정 후 제출 비율 비교. 수정 폭(±%) 분포 분석.
성공 기준
AI 산출값 수용률 ≥ 50% (수정 없이 진행)
수정 시 평균 편차 ≤ ±30%
H3
습관 형성 가설 — 한 번 쓴 사용자가 다음 달에도 돌아온다
위험도 MID
가설 내용
첫 피치덱을 생성한 사용자가 "다음 달 성과도 기록해야겠다"는 지속 사용 동기를 갖는다.
검증 방법
30일 리텐션(D30 Retention) 추적. 2회 이상 피치덱 생성 유저 비율 측정.
성공 기준
D30 Retention ≥ 25%
2회 이상 생성 유저 비율 ≥ 20%
H4
바이럴 가설 — 피치덱을 만든 사용자가 지인에게 자발적으로 공유한다
위험도 MID
가설 내용
"나도 이걸로 연봉 협상 준비했어"라는 소셜 공유 및 추천 행동이 자연 발생한다. End-User가 채널 역할을 한다.
검증 방법
신규 가입자의 유입 경로(Referral 비율) 추적. 공유 버튼 클릭률 & 실제 공유 후 가입 전환 수 측정.
성공 기준
Referral 유입 비율 ≥ 20%
NPS(Net Promoter Score) ≥ 30
H5
B2B WTP 가설 — 경영진이 전사 ROI 대시보드에 돈을 낼 의향이 있다
위험도 LOW (검증 후 개발)
가설 내용
실무자의 피치덱을 받아본 경영진이 "이 데이터를 전사적으로 보고 싶다"는 지불 의사(WTP)를 표현한다.
검증 방법
경영진 랜딩페이지 사전 신청(Waitlist) 전환율. "얼마를 내겠는가?" 가격 질문 응답 수집.
성공 기준
Waitlist 신청 ≥ 10건 (3개월 내)
제시 가격 중앙값 ≥ 월 3만원/seat

Phase별 검증 타임라인
Phase 0 · 배포 전 (D-7 ~ D0)
MVP 빌드 완료 & 배포 준비
단일 기능(자유 텍스트 입력 → AI 환산 → 피치덱 PDF) 완성. 이벤트 트래킹 설정(가입, 이슈 입력, PDF 완성, 재방문). 채널별 UTM 링크 생성.
H2 준비 (AI 품질) 이벤트 트래킹 설정 UTM 링크 세팅
Phase 1 · Week 1~2
문제 & 기능 가설 동시 검증 (H1 + H2)
블라인드 아티클 + 단톡방 무료 리포트 배포. 랜딩→가입 CVR 및 이슈 입력 시작률 측정. AI 산출값 수용률 및 수정 폭 분석. 이탈 지점 히트맵 분석.
H1 검증: CVR ≥ 15%? H2 검증: AI 수용률 ≥ 50%? 이탈 분석
Phase 2 · Week 3~4
Activation 깊이 검증 & 첫 번째 피드백 루프
첫 피치덱 완성률(≥ 40%) 확인. H1/H2 기준 미달 시 즉각 프롬프트 개선 또는 UX 수정. D7 Retention 선행 측정 시작. 피치덱 완성자 대상 짧은 인앱 설문(3문항) 진행.
Activation ≥ 40%? D7 Retention 선행 관찰 인앱 설문 3문항
Phase 3 · Month 2
습관 & 바이럴 가설 검증 (H3 + H4)
D30 Retention(≥ 25%) 및 Referral 유입 비율(≥ 20%) 측정. 피치덱 공유 버튼 클릭률 추적. NPS 측정. Referral 루프가 형성되면 채널 스케일업 판단.
H3: D30 ≥ 25%? H4: Referral ≥ 20%? NPS ≥ 30? 채널 스케일업 판단
Phase 4 · Month 3~6
B2B WTP 검증 & 수익 모델 확인 (H5)
피치덱 하단에 "경영진 대시보드" 티저 삽입. Waitlist 신청 ≥ 10건 달성 시 B2B 대시보드 개발 착수. Free→Paid CVR ≥ 3% 달성 시 B2C 수익화 강화. 두 지표 모두 미달 시 가격/기능 조합 재설계.
H5: Waitlist ≥ 10건? Free→Paid ≥ 3%? B2B 개발 착수 판단
비상구 · 언제든
인터뷰 전환 트리거
H1 미달(CVR < 10%) 또는 H2 미달(AI 수용률 < 30%)이 2주 이상 지속될 경우, "MVP 우선" 전략을 중단하고 인터뷰 5인 이상을 즉각 진행하여 가설 재설정.
CVR < 10% 2주 지속 AI 수용률 < 30% → 인터뷰 5인 전환

의사결정 기준

지표 결과에 따른 다음 액션 — 데이터가 말하는 대로 움직인다

🔵 H1 결과 — 랜딩 → 가입 CVR이 ≥ 15%인가?
✅ YES
문제 가설 1차 통과. H2(AI 품질) 검증으로 즉시 이동. 채널 확장(추가 커뮤니티 배포) 병행 시작.
❌ NO (< 10%)
랜딩 페이지 메시지 A/B 테스트 시작. 2주 내 미개선 시 인터뷰 5인 전환. 타겟 채널 재검토.
🟣 H2 결과 — AI 산출값 수용률이 ≥ 50%인가?
✅ YES
핵심 기술 가설 통과. 프롬프트 유지하며 엣지 케이스(직군별 편차) 고도화에 집중. Activation 퍼널 개선으로 이동.
❌ NO (< 30%)
즉각 프롬프트 재설계 스프린트 1주. 수정 로그를 분석해 어떤 직군·상황에서 산출 오류가 많은지 패턴 추출.
🟢 H3 결과 — D30 Retention이 ≥ 25%인가?
✅ YES
습관 형성 성공. 채널 스케일업 + 리퍼럴 루프 강화에 집중. 유료 전환 실험(심층 리포트 가격 테스트) 시작.
❌ NO (< 15%)
리마인더 / 월간 성과 기록 넛지 기능 추가. "연봉 협상 시즌"(11~1월) 타겟 리인게이지먼트 캠페인 설계.
🟡 H4 결과 — Referral 유입 비율이 ≥ 20%인가?
✅ YES
바이럴 루프 작동. 공식 추천인 프로그램(Referral 보상) 도입으로 루프를 구조화. CAC 하락 확인 후 투자 집행 검토.
❌ NO (< 10%)
피치덱 완성 시 공유 버튼 UX 강화. "이 리포트 만들어준 서비스" 워터마크 삽입. 단톡방 배포용 카드뉴스 포맷 제공.
