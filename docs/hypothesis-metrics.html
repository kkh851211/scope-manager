가설 및 성공 지표 설정
Scope Manager의 핵심 가설을 정의하고, MVP 행동 데이터로 각 가설을 검증하기 위한 성공 지표를 설계했습니다.
인터뷰가 아닌 실제 사용자 행동이 검증의 기준입니다.
⚡ 검증 전략의 전제

전통적 린 스타트업은 인터뷰 → 문제 검증 → 솔루션 검증 → MVP 순서를 따른다. 그러나 AI 개발 도구로 MVP 비용이 극적으로 낮아진 지금, MVP를 먼저 배포하고 행동 데이터로 가설을 검증하는 역전 전략이 더 빠르고 정확하다. 말(intention)이 아닌 행동(behavior)이 진실을 말한다.

가설 계층 구조
  • H1 문제 가설 — PM들이 실제로 추가 비용 청구를 포기한다
  • H2 솔루션 가설 — 시스템 판정 리포트가 AHA 모멘트를 만든다

검증 방법론
  • 1차 신호 — MVP 행동 데이터 (활성화·이탈 패턴)
  • 2차 신호 — 리포트 생성 후 Paid 전환 여부
  • 3차 신호 — 재방문율 및 두 번째 프로젝트 등록
  • 보완 수단 — 이탈 구간 인터뷰 (선택적·소수)

각 가설은 "우리가 옳다면 이런 행동이 관찰될 것이다"는 형식으로 정의됩니다.
가설이 틀렸을 때의 피벗 방향도 함께 설계했습니다.
H1 — 문제 가설 P1 최우선 검증 대기
PM들은 추가 비용 청구를 구조적으로 포기하고 있다
가설 문장: 5~20인 웹 에이전시 PM의 60% 이상은 클라이언트의 추가 요구 발생 시 비용 청구를 포기한 경험이 있으며, 포기의 주요 이유는 "증거 부족"이 아닌 "관계 악화 우려"다. 이를 '시스템 판정'으로 대체하면 심리적 장벽이 낮아진다.

성공 지표 — AARRR 프레임워크
Acquisition(획득) → Activation(활성화) → Retention(유지) → Revenue(수익) → Referral(추천)
각 단계별 지표와 목표값, 그리고 측정 방법을 정의합니다.
A1 — Acquisition · 획득
얼마나 많은 잠재 고객이 유입되는가
주 채널: SEO + 커뮤니티
월간 방문자목표 500명

사용자가 핵심 가치를 처음 경험하는가 (AHA 모멘트)

첫 프로젝트 등록률목표 40%

6주 타임라인
  • W1
    MVP 배포 + 첫 유입
    랜딩페이지 라이브, SEO 기본 세팅, 커뮤니티 소개 글 게시.
    핵심 질문: 실제로 가입하는 사람이 있는가?
    목표: 가입자 5명
  • W2
    H1 문제 가설 검증
    온보딩 설문 데이터 분석. PM들이 실제로 청구를 포기하는지 확인.
    핵심 질문: 포기 경험률이 60%를 넘는가?
    판단 기준: 60% 미달 시 문제 재정의
  • W3
    H2 솔루션 가설 검증 (AHA 관찰)
    리포트 생성 코호트 vs 미생성 코호트 재방문율 비교.
    핵심 질문: 리포트 생성이 재방문을 유발하는가?
    판단 기준: 생성군 재방문율 > 40%
  • W4
    H3 수익 가설 검증 (페이월 오픈)
    유료 플랜 전환 데이터 수집. 이탈 유저 대상 소수 인터뷰 실시.
    핵심 질문: 29,000원에 실제로 결제하는가?
    판단 기준: 전환율 > 5%
  • W5
    리텐션 심화 분석
    2주 재방문율 측정, 이탈 구간 히트맵 분석, 온보딩 개선 적용.
    핵심 질문: 한 번 쓴 유저가 계속 돌아오는가?
    판단 기준: 14일 재방문율 > 30%
  • W6
    종합 판단 + 다음 단계 결정
    4개 가설 검증 결과 종합. 지속/피벗/스톱 의사결정 실행.
    핵심 질문: 어떤 가설이 확인/기각됐는가?
    산출물: 6주 검증 리포트
6주 후 의사결정 기준
✅ 계속 (Continue)
H1·H2 모두 통과 + 유료 전환자 2명 이상 + 이탈율 < 70%
🔄 피벗 (Pivot)

⛔ 스톱 (Stop)
H1 실패 (포기 경험률 < 30%) → 문제 가설 자체를 재정의
6주 이후 검증할 오픈 이슈
  • 클라이언트 수용도 — 시스템 판정 결과를 클라이언트가 받아들이는가?
  • 기존 도구 위협 — Jira/Notion이 이 기능을 추가할 경우 차별점 유지 가능한가?
  • 국제화 가능성 — 동일 문제가 해외 에이전시에도 존재하는가?
  • AI 자동화 한계 — 범위 판정 정확도가 실무 신뢰를 얻을 수준인가?
