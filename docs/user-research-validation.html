왜 인터뷰 대신 MVP인가

사용자 리서치 전략의 패러다임 전환 — 말 데이터 → 행동 데이터

❌ 기존 방식 — 인터뷰 우선
인터뷰 섭외·일정 조율에 수 주 소요
"쓸 것 같다"는 말이 실제 사용으로 이어지지 않는 False Positive
실물 없이 진행 → 인터뷰이가 자신의 니즈를 정확히 언어화 못함
분석 → 해석 → 문서화의 주관적 필터 개입
AI 시대에 인터뷰 비용 > MVP 개발 비용으로 역전
VS
✅ 선택 방식 — MVP 즉시 배포
배포 후 즉시 실데이터 수집 시작
가입·이탈·완성 등 행동은 거짓말하지 않음
실물을 보여주고 반응을 관찰 → 구체적 개선 포인트 도출
이벤트 트래킹으로 객관·반복 측정 가능
AI 도구로 단일 기능 MVP를 수 주 내 배포 가능
이 전략이 유효한 3가지 전제 조건
🤖
MVP 개발 비용의 역전
소급 텍스트 입력 + AI 환산 + 피치덱 생성이라는 단일 기능으로 범위를 좁혀, AI 도구만으로 수 주 내 배포 가능한 수준으로 설계 완료.
🎯
명확한 얼리어답터 채널
블라인드·직군 단톡방 등 타겟 실무자 집결지가 이미 존재. 배포 직후 초기 트래픽 확보 경로 확보.
📏
이진(Binary) 측정 가능성
"피치덱을 완성했는가/안 했는가"처럼 Yes/No로 끊리는 행동 지표 존재. 해석 모호성 없이 가설 판정 가능.
인터뷰를 완전히 버리지 않는 이유 — 비상구(Fallback)
MVP 우선 전략의 한계 인정: 행동 데이터가 "무엇이 잘못됐는지(What)"는 알려줘도 "왜 잘못됐는지(Why)"는 말해주지 않는다. 따라서 핵심 지표(CVR < 10%, AI 수용률 < 30%)가 2주 이상 기준 미달일 경우, 즉각 인터뷰 5인 이상으로 전환해 "왜" 이탈하는지를 탐색한다. MVP는 인터뷰를 없애는 게 아니라 인터뷰가 필요한 시점을 뒤로 미루고, 더 구체적인 질문을 들고 인터뷰에 입장하는 전략이다.
인터뷰 전환 트리거랜딩→가입 CVR < 10% 2주 지속 또는 AI 산출값 수용률 < 30%
인터뷰 대상가입 후 피치덱을 완성하지 못하고 이탈한 실무자 5인 이상
핵심 질문"어느 단계에서 멈췄나요?" — MVP가 정확한 이탈 지점을 먼저 알려주기 때문에 질문이 훨씬 구체적
목표행동 데이터가 발견한 문제의 '왜'를 인터뷰로 확인 후, 프롬프트·UX 재설계 착수

핵심 페인 시나리오

MVP가 해결하려는 4가지 구체적 상황 — 린 캔버스 문제 블록의 행동 번역

P1
사전 예방의 투명화 — "사고가 안 났으니 아무것도 안 한 줄 안다"
강도 HIGH
상황
일정 지연을 막기 위해 유관 부서를 설득하고 요구사항을 미리 보완했다. 결과적으로 사고가 발생하지 않았다. 시스템에는 아무 기록도 남지 않는다.
페인 포인트
"이번엔 무난했네"라는 한마디에 치열했던 리스크 헷징이 투명화된다. 증명할 데이터가 없다.
MVP 검증 포인트
이 상황의 실무자가 "사전 예방" 유형 이슈를 AI 입력창에 기술하는지, AI가 이를 금액으로 변환하는 품질이 납득 가능한지.
P2
소방수 딜레마 — "위기를 수습했더니 오히려 본업 지연으로 감점"
강도 HIGH
상황
돌발 위기에 투입되어 회사를 구했지만, 공식 평가에는 본업 일정이 지연된 것만 기록된다. 기여는 사라지고 감점만 남는다.
페인 포인트
예방도 수습도 할 동기를 상실. 방어적 업무 태도로 전환. 가장 유능한 사람이 가장 불합리한 평가를 받는 구조.
MVP 검증 포인트
위기 수습 사례를 텍스트로 기술했을 때 AI가 "회사에 대한 재무적 기여"로 재프레이밍하는 품질 — 이것이 핵심 Aha! Moment.
P3
빈틈 메우기의 비극 — "본업 외 헌신이 '잡무'로 평가절하"
강도 MID
상황
기획자가 CS를 처리하고, 마케터가 영업을 지원하며 조직의 싱크홀을 메웠다. 연말 평가는 "그래서 본업 성과는 뭔데?"라고 묻는다.
페인 포인트
회사를 굴러가게 한 헌신이 '잡무'로 전락. 역할 중첩(Role Overlap) 기여를 표준 지표로 환산하는 방법이 없다.
MVP 검증 포인트
"역할 외 기여" 유형의 텍스트가 AI에 의해 조직 기여 금액으로 변환되는지. 직군별 편차 분석 필요.
P4
연봉 협상 무기 부재 — "증거 없이 감이나 관계로만 협상해야 한다"
강도 MID
상황
연봉 협상 시즌, 실제로 만들어낸 성과는 많지만 데이터로 정리된 문서가 없다. 감이나 관계에 의존하거나 급조한 PPT로 임한다.
페인 포인트
협상 테이블에서 근거 없는 요구로 보인다는 심리적 열위. "내가 얼마를 만들었는지" 재무 언어로 말하지 못하는 답답함.
MVP 검증 포인트
완성된 피치덱이 "이걸 가져가면 협상이 달라지겠다"는 확신을 주는지 — PDF 다운로드 후 재방문률로 간접 측정.

행동 신호 정의

인터뷰 응답 대신 측정할 사용자 행동 이벤트 사전(事典)

End-User (실무자) 핵심 이벤트 & 해석
이벤트정의 & 측정 방법신호 유형해석
landing_visit
랜딩 페이지 최초 방문
 채널별 UTM 파라미터로 유입 경로 구분 (블라인드/단톡방/검색) 기준점 채널 효율 비교 기준. 절댓값보다 채널별 CVR 차이가 중요.
signup_complete
회원가입 또는 소셜 로그인 완료
 랜딩→가입 CVR = signup / landing_visit H1 검증 ≥ 15% → 문제 가설 1차 통과. 랜딩 메시지가 페인을 정확히 건드림.
issue_input_start
이슈 입력창 첫 글자 타이핑
 가입 후 입력 시작까지의 Time-to-Action 측정 진입 신호 "지금 당장 쓸 내용이 있다" = 긴급 페인 보유 확인.
issue_input_3_complete
이슈 3건 입력 완료 후 AI 생성 요청
 3건 모두 작성 후 "분석하기" 버튼 클릭 H2 선행 입력 1~2건에서 이탈 시 → 입력 UX 마찰 또는 "쓸 내용이 없다" 의미.
ai_result_accepted
AI 산출 금액을 수정 없이 그대로 채택
 금액 수정 없이 다음 단계 진행한 비율 H2 핵심 ≥ 50% → AI 프롬프트 품질 통과. 핵심 기술 가설 검증.
ai_result_edited
AI 산출 금액 또는 설명 수정
 수정 전후 금액 차이(±%) 및 텍스트 변경량 기록 H2 보조 평균 편차 > ±30% → 프롬프트 재설계 필요. 어떤 직군에서 편차가 큰지 분류.
pitchdeck_generated
피치덱 PDF 생성 완료 🎯
 가입자 대비 완성률 = pitchdeck_generated / signup_complete Activation ≥ 40% → 핵심 Aha! Moment 도달. 이 지표가 North Star Metric.
pitchdeck_downloaded
PDF 다운로드 또는 공유 링크 복사
 생성 후 다운로드까지 Time-to-Download, 재다운로드 횟수 사용 의지 재다운로드 = "실제로 협상에 가져감". 가장 강력한 가치 증명 신호.
d7_return / d30_return
7일/30일 후 재방문 로그인
 코호트 분석. 가입일 기준 D7, D30 활성 세션 여부 H3 검증 D30 ≥ 25% → 습관 형성 성공. D7 < 15% = D30 악화 선행 신호.
referral_share
공유 버튼 클릭 또는 추천 링크 생성
 공유 이벤트 발생 비율 + 공유 후 신규 가입 전환 수 H4 검증 "나도 이걸로 협상했어" = 자발적 바이럴 루프 시작 신호.
mid_funnel_exit
이슈 1건 입력 후 이탈
 입력 1건 이후 세션 종료, 재방문 없음 위험 신호 > 50% → UX 마찰 또는 AI 결과 품질 실망. 즉각 프롬프트·인터페이스 점검.
B2B (경영진) 핵심 신호
🟢 WTP 신호
경영진 랜딩 방문 후 Waitlist 신청
실무자 피치덱 하단 링크 클릭 → 대시보드 소개 페이지 도달 → 신청 완료. 3개월 내 10건 = H5 가설 통과. B2B 개발 착수 트리거.
🟡 관심 신호
랜딩 방문 후 이탈 (신청 없음)
관심은 있으나 WTP 미확인. 가격 제시 방식 또는 가치 설명 방식 변경 필요. 방문 시간 > 2분이면 재방문 리타겟팅 시도.
📩 가격 신호
Waitlist 신청 시 "희망 가격" 응답
선택형 설문으로 수집. 중앙값 ≥ 월 3만원/seat = 수익 모델 성립 확인. 분포를 보고 Per-seat vs. 기업 단위 과금 방식 결정.
🔴 무관심 신호
피치덱 링크 클릭 없음
실무자가 피치덱을 제출했으나 경영진이 링크를 클릭하지 않음. 피치덱 포맷 또는 링크 노출 방식 재설계 필요. B2B 채널 전략 전환 검토.

검증 매트릭스

린 캔버스 각 블록별 검증 질문 → 측정 방법 → 판정 기준 전체 매핑

블록별 핵심 검증 질문 매핑
캔버스 블록 검증 질문 측정 방법 (행동 데이터) 성공 기준 검증 시점
Block 1
문제		 실무자의 '증명 불가' 페인이 행동 전환을 일으킬 만큼 강렬한가? 랜딩→가입 CVR, 첫 이슈 입력 시작률 CVR ≥ 15% Week 1~2
Block 4
솔루션		 AI 텍스트→금액 변환 결과가 실무자에게 납득 가능한 수준인가? AI 산출값 수용률, 수정 폭 분포, 이슈 타입별 만족도 수용률 ≥ 50% Week 1~2
Block 3
UVP		 피치덱 완성이 실무자에게 "협상 무기"라는 확신을 주는가? PDF 다운로드율, 재다운로드 횟수, D30 재방문율 완성률 ≥ 40% Week 3~4
Block 2
고객군		 이직/협상 준비 중인 PM·기획자가 가장 빠르게 전환하는가? UTM별 채널 CVR 비교, 가입 시 직군·상황 선택 데이터 채널별 CVR 비교 Week 1~4
Block 8
지표		 D30 재방문이 WAU 30% 목표로 이어지는 습관이 형성되는가? 코호트별 D7/D30 Retention, 월간 활성 이슈 기록 수 D30 ≥ 25% Month 2
Block 5
채널		 블라인드·단톡방 바이럴이 Referral 루프를 형성하는가? 신규 가입자 유입 경로 비율, Referral 추천→가입 전환 수 Referral ≥ 20% Month 2
Block 6
수익원		 무료 사용자가 심층 리포트 유료 기능에 지불 의사를 갖는가? 유료 버튼 클릭률, Free→Paid CVR, ARPU CVR ≥ 3% Month 3~6
Block 6
B2B		 경영진이 전사 ROI 대시보드에 실제 돈을 낼 의향이 있는가? Waitlist 신청 건수, 희망 가격 응답 분포 Waitlist ≥ 10건 Month 3~6
Block 9
경쟁우위		 AI 변환 품질이 사용자가 직접 만드는 것보다 명백히 낫다고 느끼는가? AI 수용률, 인앱 설문 "AI 결과 만족도", NPS NPS ≥ 30 Month 2~3

학습 루프 설계

행동 데이터 → 인사이트 추출 → 제품 개선의 반복 사이클

Build-Measure-Learn 사이클 — MyROI 맞춤형
BUILD
단일 기능 MVP 배포 (소급 텍스트 입력 → 피치덱)
이메일 파싱 등 복잡한 기능을 제거하고 핵심 가치 전달의 최소 단위만 구현. 이벤트 트래킹 코드 삽입 완료. 채널별 UTM 링크 생성.
단일 기능 집중 이벤트 트래킹 설정 A/B 테스트 기반 세팅
MEASURE
행동 이벤트 수집 & 퍼널 분석
랜딩→가입→입력→AI결과→피치덱 완성의 각 단계 이탈률을 매일 모니터링. AI 수용/수정 패턴을 직군별·이슈 유형별로 세분화. D7 선행 지표로 D30을 예측.
일별 퍼널 모니터링 직군별 AI 수용률 분류 이탈 지점 히트맵
LEARN — What
행동 데이터가 "무엇이 문제인지" 알려준다
이탈 지점 = 마찰의 좌표. AI 수정 폭이 큰 직군 = 프롬프트 개선 우선순위. D7 낮은 코호트 = 습관화 실패 패턴. 데이터가 먼저 문제를 가리키면, 인터뷰는 "왜"를 확인하는 역할로 축소된다.
이탈 지점 특정 직군별 편차 발견 코호트 패턴 추출
LEARN — Why (선택적)
인터뷰는 "왜"가 필요할 때만, 더 좋은 질문을 들고 진행
MVP 없이 진행하는 맹목적 인터뷰와 달리, 이미 이탈 지점을 알고 있는 상태에서 인터뷰를 진행. "3번째 이슈 입력 후 이탈하셨는데, 어느 부분이 막히셨나요?"처럼 핀포인트 질문 가능. 인터뷰의 신뢰도가 획기적으로 높아진다.
이탈자 대상 핀포인트 인터뷰 피치덱 완성자 NPS 설문 인앱 마이크로 설문 (3문항)
ITERATE
1주 단위 스프린트로 가설 → 측정 → 반영 반복
프롬프트 개선, UX 수정, 랜딩 메시지 변경을 1주 스프린트로 배포 후 동일 지표 재측정. 지표 개선이 없으면 가설 자체를 교체. 2회 이상 스프린트 후 개선 없음 = 피벗 신호로 처리.
1주 스프린트 A/B 테스트 2회 미개선 → 피벗 검토
데이터 기반 적응 시나리오 — 지표가 이렇게 나오면 이렇게 한다
🔴
시나리오 A — 랜딩 CVR < 10% (문제 가설 미통과)
가입 전환이 너무 낮다 → 메시지가 페인을 건드리지 못하고 있다
즉각 액션 (1주 내)
랜딩 헤드라인 A/B 테스트. "연봉 협상" vs "숨은 성과 증명" 중 어떤 프레임이 더 높은 CVR을 만드는지 비교.
2주 후에도 미개선 시
인터뷰 5인 즉각 전환. 타겟 채널 자체를 재검토. 블라인드가 아닌 LinkedIn 직접 DM 시도.
🔴
시나리오 B — AI 수용률 < 30% (기술 가설 미통과)
AI 산출값을 신뢰하지 않는다 → 프롬프트 품질 위기
즉각 액션 (1주 내)
수정 로그 분석 → 어떤 직군·이슈 유형에서 편차가 가장 큰지 추출 → 해당 케이스 중점 프롬프트 재설계 스프린트.
보완 방안
AI 산출 결과에 산정 근거 투명화(계산식 노출) 추가. "이 금액이 너무 낮거나 높으신가요?" 인앱 피드백 수집.
🟡
시나리오 C — Activation ≥ 40% but D30 < 20% (습관화 실패)
처음엔 쓰지만 돌아오지 않는다 → 지속 사용 동기 부재
즉각 액션
월간 리마인더 이메일 도입. "지난달에 방어한 성과, 이달에도 기록하시겠어요?" — 연봉 협상 시즌(11~1월) 타이밍 넛지.
가설 재설정
피치덱은 일회성 협상 도구일 수 있음. WAU보다 "연봉 협상 시즌 활성률"을 핵심 지표로 재정의할지 검토.
🟢
시나리오 D — Activation ≥ 40% + D30 ≥ 25% (지속 신호)
핵심 가설 통과 → 스케일업 집중 단계
즉각 액션
채널 스케일업 (블라인드 유료 광고, LinkedIn 확장). Referral 프로그램 공식화 (추천 시 리포트 1회 무료 제공).
