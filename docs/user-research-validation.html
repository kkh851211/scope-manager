Scope Manager의 핵심 타겟 사용자를 정의하고, 그들이 겪는 문제를 구조적으로 분석합니다.
전통적 인터뷰 대신 MVP 행동 데이터를 1차 검증 수단으로 삼는 역전 전략을 채택했습니다.
⚡ 전략적 선택 — 인터뷰 없이 MVP로 직행

AI 개발 도구의 발달로 MVP 제작 비용이 인터뷰 설계·진행 비용보다 낮아졌다. 말(intention)과 행동(behavior)은 다르다. 실제 사용·이탈 패턴이 가장 정직한 사용자 리서치다. "만들어서 보여주는" 것이 "물어보는" 것보다 빠르고 정확한 검증이다.

검증 전략 — 인터뷰 vs MVP 행동 데이터
왜 고객 인터뷰를 생략하고 MVP로 직행하는가? 전략적 근거와 실행 설계를 정리합니다.
전통적 린 스타트업 방법론
  • 고객 인터뷰 설계 (2~4주)
  • 인터뷰 대상자 모집 (1~2주)
  • 인터뷰 진행 (10~20명)
  • 데이터 분석 및 인사이트 도출
  • 문제 검증 완료 후 솔루션 검증
  • 솔루션 검증 후 MVP 개발 착수
  • 총 소요 기간: 8~16주
→
AI 시대 역전 전략 (채택)
  • AI 도구로 MVP 즉시 개발 (1~2주)
  • 랜딩페이지 + 온보딩 설문 설계
  • MVP 배포 + 실제 사용자 유입
  • 행동 데이터 수집 (사용/이탈 패턴)
  • 필요 시 이탈 유저 5명 인터뷰
  • 데이터 기반 피벗 또는 확장 결정
  • 총 소요 기간: 3~6주
전략 전환의 4가지 논리
  • 말 ≠ 행동 — "쓸 것 같다"(인터뷰)와 "실제로 쓴다"(행동)는 다르다. 행동 데이터가 더 정직하다.
  • 비용 역전 — AI 덕분에 MVP 개발 비용 < 인터뷰 설계·진행 비용. 경제성 역전이 일어났다.
  • 속도 우위 — 인터뷰에 8~16주를 쓰는 대신 3~6주 만에 실제 데이터를 확보할 수 있다.
  • 정확성 우위 — 10명의 말보다 100명의 행동이 더 강력한 검증이다.
인터뷰를 "선택적"으로 유지하는 이유
  • 행동 데이터 한계 — 이탈 이유는 행동 데이터만으로 설명되지 않을 수 있다.
  • 소수 정예 인터뷰 — 이탈 구간 특정 후 5명 이내 텍스트 인터뷰로 "왜"를 보완한다.
  • 타이밍 최적화 — 인터뷰를 "전"이 아닌 "후"에 배치하여 더 정확한 질문을 설계한다.
  • 신뢰도 보완 — 행동 데이터로 검증된 가설을 인터뷰로 재확인하는 순서.

이탈 유저 인터뷰 (W3~W4)
  • 대상 — 가입 후 72시간 내 프로젝트 미등록 이탈자 5명
  • 방법 — 이메일 → 카카오톡 텍스트 인터뷰 (비동기)
  • 핵심 질문 — "첫 번째 단계에서 막힌 이유가 뭔가요?"
  • 목표 — 온보딩 UX 개선 포인트 도출
AHA 경험자 인터뷰 (W4~W5)
  • 대상 — 리포트 생성 완료 후 재방문한 유저 3명
  • 방법 — 30분 줌 인터뷰 또는 텍스트
  • 핵심 질문 — "리포트를 실제로 클라이언트에게 보냈나요?"
  • 목표 — 실제 사용 맥락과 가치 체감 확인
Paid 전환자 인터뷰 (W6 이후)
  • 대상 — Starter 결제 완료 유저 전원 (초기)
  • 방법 — 이메일 감사 메시지 + 1개 질문
  • 핵심 질문 — "결제를 결심한 순간이 언제였나요?"
  • 목표 — 구매 트리거 파악 → 마케팅 메시지 최적화
