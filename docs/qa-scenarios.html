🎯 가설 검증 우선

인터뷰 대신 행동 데이터로 가설 검증. AC는 "기능이 동작하는가"를 넘어 "트래킹 이벤트가 올바르게 발화하는가"까지 포함한다.

✅ Activation 완주 보장

배포 Go / No-Go 기준
🟢 Go — 배포 가능
  • P0 시나리오 (QA-HP-01~09) 전체 PASS
  • 트래킹 이벤트 P0 3종 발화 확인
  • AI 응답 시간 p95 < 15초
  • PDF 생성 < 5초
🟡 조건부 Go — 이슈 트래킹 후 배포
  • P1 시나리오 FAIL 1건 이하 (Critical 아닌 경우)
  • 엣지케이스 이슈는 Hotfix 티켓 등록 후 배포
🔴 No-Go — 배포 불가
  • P0 시나리오 어느 하나라도 FAIL
  • 트래킹 이벤트 미발화 (가설 검증 불가)
  • AI 결과 JSON 파싱 오류 발생
  • PDF 생성 완전 실패

인수 조건 (Acceptance Criteria)

Given / When / Then 형식 · 트래킹 이벤트 연동 · 클릭하여 상세 펼치기

AC-001 자유 텍스트 이슈 입력 P0 · F-001 ▼
GIVEN사용자가 로그인 후 이슈 입력 화면에 진입했다
WHEN자유 텍스트 필드에 입력을 시작한다
THEN실시간 글자 수 카운터가 업데이트된다
AND직무에 맞는 예시 힌트가 텍스트 아래에 표시된다
ANDissue_input_started 이벤트가 발화된다
✅
20자 이상 입력 시 "AI 분석 시작" 버튼이 활성화된다
🚫
19자 이하 시 버튼이 비활성화되고 "좀 더 구체적으로 적어주세요" 인라인 문구 표시
✅
500자 도달 시 추가 입력이 차단되고 카운터가 빨간색으로 변경된다
✅
이슈 2·3번 필드는 접힌 상태로 시작하며, 이슈 1 입력 후 "+ 이슈 추가" 버튼이 나타난다
✅
페이지 새로고침 전까지 입력 내용이 유지된다 (세션 임시 저장)
⚠️
"예시로 바로 시작하기" 클릭 시 예시 텍스트가 필드에 자동 삽입되고 example_used: true 속성이 기록된다
트래킹: issue_input_started · issue_input_completed · ai_analysis_triggered
AC-002 AI 이슈 구조화 + 재무 금액 산출 P0 · F-002 ▼
GIVEN사용자가 20자 이상의 이슈 텍스트를 입력하고 "AI 분석 시작"을 클릭했다
WHENLLM API 호출이 완료된다
THEN카테고리, 예상 금액(원), M/M 환산, 신뢰도(%), 근거 요약이 화면에 표시된다
ANDai_result_displayed 이벤트가 모든 속성 값과 함께 발화된다
AND로딩 화면에서 단계별 진행 상태가 시각적으로 업데이트된다
✅
AI 응답이 유효한 JSON 형식이며 required 필드(category, estimated_amount_krw, confidence_pct, reasoning)를 모두 포함한다
✅
예상 금액이 0원 초과, 100억원 이하의 유효 범위 내에 있다
✅
신뢰도 50% 미만 시 "입력을 더 구체화하면 정확도가 높아집니다" 안내 배너가 노출된다
🚫
15초 초과 타임아웃 시 에러 상태로 전환되고 "재시도" 버튼이 표시된다 (로딩 무한 루프 없음)
🚫
JSON 파싱 실패 시 사용자에게 "분석 중 오류가 발생했습니다" 메시지를 표시하고 ai_analysis_failed 이벤트를 발화한다
⚠️
금액 범위(min~max)가 화면에 표시되어 AI 산출의 불확실성을 시각적으로 전달한다
트래킹: ai_result_displayed · ai_analysis_failed
AC-003 Human-in-the-loop 금액 검토 & 수정 P0 · F-003 ▼
GIVENAI 결과 화면에서 금액이 표시된 상태이다
WHEN사용자가 금액 필드를 직접 수정하거나 +/- 스텝 버튼을 클릭한다
THEN수정된 금액이 즉시 화면에 반영된다
AND원래 AI 산출 금액과 수정 금액이 나란히 표시된다 (Before/After)
ANDamount_edited 이벤트에 was_edited:true, edit_direction, edit_pct가 기록된다
✅
수정 가능 범위: AI 산출 금액의 ±300% 이내. 범위 초과 입력 시 "현실적인 범위를 벗어났습니다" 확인 팝업 표시
🚫
0원 또는 음수 입력 불가 — 입력 시 즉시 최소값(100만원)으로 자동 보정
✅
이미지 첨부 시 파일 크기(5MB 초과), 형식(jpg/png/webp/pdf만 허용) 검증 후 에러 메시지 표시
✅
첨부 이미지는 최대 3개까지 허용, 4번째 추가 시도 시 "최대 3개까지 첨부 가능합니다" 안내
✅
"확정하기" 클릭 시 수정하지 않은 경우 was_edited:false로 기록되어 AI 정확도 추적에 활용
트래킹: amount_edited · evidence_attached · issue_confirmed
AC-004 연봉 피치덱 PDF 원클릭 생성 P1 · F-004 ▼
GIVEN하나 이상의 이슈가 확정 상태이고 사용자가 "피치덱 생성" 버튼을 클릭했다
WHENPDF 렌더링이 완료된다
THEN미리보기 화면에서 PDF 레이아웃이 표시되고 다운로드 / 링크 공유 버튼이 활성화된다
ANDpitchdeck_generated 이벤트에 is_first_pitchdeck, total_amount_krw, time_to_activate_min이 기록된다
✅
PDF에 사용자 이름, 직무, 생성 날짜, 각 이슈(카테고리/설명/금액), 총합, 면책 문구가 모두 포함된다
✅
PDF 생성 시간 5초 이내 (p95 기준)
⚠️
월 1회 무료 한도 초과 시 "이번 달 무료 생성 횟수를 사용했습니다" 팝업 + 유료 플랜 유도 CTA 표시. upgrade_prompt_shown 이벤트 발화.
✅
링크 공유 시 발급된 URL이 로그인 없이 열람 가능한 읽기 전용 페이지로 연결된다
🚫
이슈가 0개인 상태에서 "피치덱 생성" 버튼 클릭 시 버튼이 비활성화되어 있거나 "먼저 이슈를 추가해주세요" 안내
트래킹: pitchdeck_generated · pdf_downloaded · link_shared · upgrade_prompt_shown
AC-005 이슈 보관함 CRUD P1 · F-005 ▼
GIVEN하나 이상의 이슈가 확정된 상태에서 보관함 화면에 진입했다
WHEN보관함이 로드된다
THEN누적 방어 금액 합계가 상단에 표시되고 이슈 목록이 최신순으로 정렬된다
ANDvault_viewed 이벤트에 total_amount_krw와 days_since_last_visit이 기록된다
✅
이슈 수정: 금액/텍스트/카테고리 변경 후 저장 시 목록과 누적 금액이 즉시 업데이트된다
🚫
이슈 삭제: 삭제 전 "삭제하면 되돌릴 수 없습니다" 확인 다이얼로그 표시 필수
⚠️
마지막 이슈 입력 후 7일 경과 시 "이번 주 새로운 이슈를 추가해보세요" 넛지 배너가 상단에 표시된다
✅
카테고리 필터 (전체/위기수습/사전방어/역할초과/지식이전/관계관리) 선택 시 해당 카테고리만 표시된다
트래킹: vault_viewed · issue_edited · issue_deleted
AC-006 온보딩 플로우 P1 · F-006 ▼
GIVEN신규 사용자가 처음으로 가입을 완료했다
WHEN가입 완료 직후 자동으로 온보딩 Step 1로 리디렉션된다
THEN4단계 프로그레스 바가 표시되고 현재 단계가 강조된다
AND직무 선택 시 해당 직무의 예시 이슈 3개가 즉시 표시된다
✅
6가지 직무(PM/기획자, 개발자, 디자이너, 마케터, 세일즈, 기타) 선택지가 모두 표시된다
✅
온보딩 중 "건너뛰기" 링크를 통해 이슈 보관함으로 바로 이동 가능 (강요 없음)
✅
온보딩 완료 플래그가 저장되어 재로그인 시 온보딩을 다시 보여주지 않는다
⚠️
브라우저 뒤로가기로 이전 온보딩 단계로 돌아갈 수 있으며 입력 내용이 유지된다
트래킹: onboarding_started · role_selected · example_used · onboarding_completed
AC-007 B2B 데모 요청 폼 P2 · F-007 ▼
GIVEN방문자가 Pricing 페이지의 B2B 플랜 "데모 요청하기" 버튼을 클릭했다
WHEN폼에 필수 항목(회사명, 직함)을 입력하고 제출한다
THEN"데모 요청이 접수되었습니다" 확인 메시지가 표시된다
ANDdemo_request_submitted 이벤트에 모든 속성이 기록된다
AND슬랙/이메일 알림이 운영팀에 발송된다
🚫
필수 항목(회사명, 직함) 미입력 시 제출 불가 + 해당 필드에 인라인 에러 표시
✅
예산 범위는 선택 항목 — 미입력 시에도 제출 가능하며 budget_range: null로 기록
✅
폼 제출 후 버튼이 "제출됨" 상태로 변경되어 중복 제출이 방지된다
트래킹: pricing_viewed · b2b_plan_clicked · demo_request_submitted
