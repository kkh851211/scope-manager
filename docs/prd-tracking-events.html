제품 목표 (MVP 4주)
Goal 1 — 핵심 가설 검증

자유 텍스트 → AI 재무 금액 변환이 "쓰고 싶다"는 욕구를 만드는가?

측정: 데모 완료율 > 40%

Goal 2 — Activation 달성

소급 입력 → 첫 피치덱 PDF 생성까지 완주 비율

측정: 가입자 중 > 50% 첫 세션 완주

Goal 3 — WAU 습관 형성

이슈 보관함 재방문으로 누적 금액 확인 루틴 형성

측정: WAU 초기 15% → 6개월 30%

Goal 4 — B2B WTP 검증

경영진이 대시보드 데이터에 실제 지불 의사를 보이는가?

측정: 데모 요청 월 10건 이상

✓LLM 기반 이슈 구조화 + 예상 방어 금액(M/M) 산출
✓Human-in-the-loop 금액 직접 수정
✓캡처/증거자료 첨부 (이미지 업로드)
✓연봉 피치덱 PDF 원클릭 생성
✓이슈 보관함 (CRUD)
✓소셜 로그인 (Google OAuth)
✓Pricing 페이지 + B2B 데모 요청 폼 (WTP 탐색)
✓트래킹 이벤트 전체 심기 (행동 데이터 수집)

✗이메일/슬랙 자동 파싱 연동 (Scope Creep 차단)

✗실시간 협업 기능 (다중 유저 공동 편집)
✗팀/조직 계정 관리 (per-seat 청구)
✗결제 시스템 (Freemium 검증 우선)
✗다국어 지원 (한국어 SME 타겟 집중)
✗HR 시스템 API 연동 (Jira/Asana 등)

F-002 AI 이슈 구조화 + 재무 금액 산출 P0 · 핵심 ▼

기능 설명LLM(GPT-4o-mini)이 이슈를 5개 카테고리로 분류 (위기수습/사전방어/역할초과/지식이전/관계관리). M/M 단가 기반 예상 방어 금액 계산. AI 신뢰도 점수(%) 함께 반환.

AI 프롬프트 설계 (Prompt Engineering)

핵심 가설 검증의 기술적 핵심 — 비정형 텍스트 → 재무 금액 변환 프롬프트

System Prompt — 이슈 분류 & 금액 산출
# Role You are a workplace contribution analyst specializing in converting informal employee contributions into quantified financial impact. Your role is to help Korean workers in SMEs (50 employees or fewer) objectively measure the value of their hidden work. # Task Analyze the user's issue description and return a structured JSON response containing: issue category, estimated defense value in KRW, M/M calculation, confidence score, and reasoning summary. # Issue Categories - crisis_response: 위기 수습 (서버 장애, 납기 위기, 계약 무산 방지) - preventive_defense: 사전 방어 (리스크 사전 차단, 재작업 방지) - role_overflow: 역할 초과 (타 직무 공백 대체, 멀티롤) - knowledge_transfer: 지식 이전 (온보딩, 문서화, 교육) - relationship_mgmt: 관계 관리 (클라이언트 유지, 갈등 중재) # M/M Unit Cost Reference (Korean SME, IT/Agency) Junior (0-3y): 3,500,000 KRW/month Mid (3-7y): 5,500,000 KRW/month Senior (7y+): 8,000,000 KRW/month Default if unknown: 5,500,000 KRW/month # Calculation Logic 1. Estimate time saved or cost avoided in person-months (M/M) 2. Multiply by appropriate M/M unit cost 3. Add any direct cost savings (contract value, error cost, etc.) 4. Confidence: HIGH(80-100%) / MED(50-79%) / LOW(<50%) # Output Format (JSON ONLY, no markdown) { "category": "crisis_response", "category_label_ko": "위기 수습", "estimated_amount_krw": 24000000, "mm_saved": 2.4, "confidence_pct": 87, "reasoning": "런칭 위기 수습으로 계약 1건 유지. 계약 규모 및 재작업 비용 기준 산출.", "amount_range": { "min": 18000000, "max": 32000000 } }

프롬프트 개선 전략 — Unfair Advantage 구축 로드맵
Phase 1 · Launch (v1)

단일 프롬프트 + 직무별 M/M 단가 테이블. 신뢰도 50% 미만 시 재입력 가이드.

측정: AI 신뢰도 평균 점수, 수정 없이 확정 비율

Phase 2 · Learning (v2)

유저 수정 데이터 누적 → 카테고리별 보정 계수 추출. 업종별 단가 테이블 세분화.

조건: 이슈 500개 이상 수집 후 적용

Phase 3 · Dataset (v3)

유저 수정 이력 → 파인튜닝 데이터셋 구축. 특화 매핑 데이터셋 선점 (Unfair Advantage).

조건: 이슈 5,000개 이상 → 경쟁사 대비 정확도 격차
