01 — Problem · 해결할 문제
핵심 문제 3가지
  • 청구 포기 구조 — "돈 달라는 말을 꺼내기 어렵다"는 심리적 장벽이 반복적 손실로 이어짐
  • AI 인식 격차 — 클라이언트는 "AI가 다 해주니 싸야 한다", 에이전시는 "AI 검증 비용이 새로 생겼다"
  • 표준 부재 — 변경 관리 단가의 업계 표준이 없어 협상이 PM 개인의 담력에 의존
기존 대안과 한계
  • 카카오톡 + 엑셀 — 기록은 있으나 판정·근거 생성 불가
  • Jira / Asana / Notion — 과업 관리 목적, 범위 초과 청구 설계 없음
  • 엔터프라이즈 CM 도구 — 고비용·복잡, 5~20인 에이전시 현실과 괴리
PMI: 프로젝트 52% 스코프 크리프 경험 예산 초과율 85% / 평균 초과 27%
02 — Solution · 솔루션
⚖️ 범위 초과 판정자
클라이언트의 요구가 계약 범위를 벗어났는지 시스템이 자동 기록·판정한다. PM의 주관이 아닌 데이터가 말한다.
📄 청구 근거 생성자
"시스템상 과업 범위를 초과한 것으로 기록되었습니다"라는 문장을 도구가 자동 생성한다. PM은 전달만 한다.
📈 수익성 가시화
프로젝트별 실제 손익을 누적 기록하여 다음 견적·계약의 기준을 데이터로 만든다.

MVP 우선 검증 전략
전통적인 린 스타트업 방법론을 AI 시대에 맞게 재설계했다.
⚡ 전략적 선택 — 고객 인터뷰 → MVP 순서 역전

AI 개발 도구의 발달로 MVP 제작 비용과 시간이 극적으로 낮아졌다. 이에 따라 불확실성이 높은 인터뷰를 생략하고 MVP를 즉시 배포하여 실제 행동 데이터로 가설을 검증하는 전략을 선택했다.

전통적 린 스타트업
  • 고객 인터뷰 설계·진행
  • 문제 검증 (말에 의존)
  • 솔루션 검증
  • MVP 개발 (고비용)
  • 배포 → 행동 데이터 수집
→
AI 시대 역전 전략
  • MVP 즉시 개발·배포 (AI 활용)
  • 실제 사용/이탈 패턴 관찰
  • 행동 데이터로 가설 검증
  • 데이터 기반 인터뷰 (선택적)
  • 검증된 기능 확장
역전 전략의 논리
  • 말 vs 행동 불일치 — 인터뷰의 intention과 실제 behavior는 다르다
  • 비용 역전 — AI 덕분에 MVP 개발 비용 < 인터뷰 설계/진행 비용
  • 신호의 정직성 — 사용/이탈 패턴이 인터뷰보다 더 정직한 검증 신호
  • 속도 우위 — "만들어서 보여주는" 것이 "물어보는" 것보다 빠름
검증할 핵심 가설
  • PM의 추가 비용 청구 포기율 > 60%
  • 시스템 판정 결과를 클라이언트가 수용한다
  • Free → Paid 전환을 유발하는 AHA 모멘트는 "첫 리포트 생성"
  • 29,000원/월 가격에 지불 의향이 존재한다
✅ MVP 검증 성공 기준 (6주 내)

회원가입 후 첫 프로젝트 등록률 > 40%  |  2주 재방문율 > 30%  |  리포트 생성 1회 이상 유저 > 20%  |  Paid 전환율 > 5%
