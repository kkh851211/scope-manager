-- 사용자는 자신의 프로젝트만 조회 가능 ALTER TABLE projects ENABLE ROW LEVEL SECURITY; CREATE POLICY "own_projects_only" ON projects FOR ALL USING (auth.uid() = user_id); -- 플랜별 프로젝트 수 제한 (Free: 1개) CREATE POLICY "free_plan_limit" ON projects FOR INSERT WITH CHECK ( get_user_plan(auth.uid()) != 'free' OR ( SELECT COUNT(*) FROM projects WHERE user_id = auth.uid() ) < 1 );
AI 범위 판정 엔진 설계
Scope Manager의 핵심 차별 기술 — Claude API를 활용한 텍스트 기반 범위 초과 자동 판정.
프롬프트 설계, 응답 구조, 정확도 개선 루프를 정의합니다.
🤖 왜 Claude API인가?

범위 초과 판정은 의미적 유사도를 기반으로 한다. "로그인 기능 추가"가 계약서의 "회원가입 모듈"에 포함되는지 판단하려면 단순 키워드 매칭이 아닌 자연어 이해가 필요하다. Claude API는 한국어 성능이 우수하고 Structured Output을 지원하며, Haiku 모델로 비용을 낮출 수 있다.

판정 프롬프트 구조
// System Prompt (고정) 당신은 웹 에이전시 PM의 프로젝트 범위 관리를 돕는 전문가입니다. 계약 범위와 추가 요구를 비교하여 정확히 3가지 중 하나로 판정하세요: - in_scope: 계약 범위 내 - out_of_scope: 범위 초과 - unclear: 판단 어려움 // User Prompt (동적 생성) ### 계약 범위 {scope_items.map(i => `- ${i.title}`)} ### 클라이언트 추가 요구 {change_request.description} ### 응답 형식 (JSON only) { "verdict": "out_of_scope", "confidence": 0.92, "reason": "판정 근거 한 문장", "related_scope": "관련 계약 항목명" }
예상 판정 시나리오 & 정확도
✓ 명확한 범위 초과 (정확도 ~90%)
계약: "회원가입 페이지" → 요구: "SNS 소셜 로그인 5종 추가"
→ out_of_scope (confidence: 0.95)
↔ 범위 내 포함 가능 (정확도 ~80%)
계약: "상품 상세 페이지" → 요구: "상품 이미지 확대 기능"
→ in_scope (confidence: 0.78)
△ 판단 불명확 (PM 확인 필요)
계약: "관리자 페이지" → 요구: "엑셀 다운로드 기능"
→ unclear → PM 수동 판정 안내
정확도 개선 피드백 루프
1단계
AI 자동 판정 → PM에게 결과 표시
2단계
PM이 판정 결과 확인/수정 (오버라이드)
3단계
수정 데이터 DB 저장 → 오류 패턴 분석
4단계
오류율 높은 케이스 → 프롬프트 개선
목표: 6주 후 판정 정확도 85% 이상
API 비용 최적화 전략
  • Haiku 기본 사용 — 판정 분류는 Haiku ($0.25/1M tokens), 리포트 생성만 Sonnet
  • 입력 토큰 압축 — 계약 범위를 bullet 형식으로 최소화 (평균 ~300 tokens/판정)
  • 캐싱 활용 — 동일 프로젝트의 시스템 프롬프트 Prompt Caching 적용
  • 배치 처리 — 여러 요구 동시 입력 시 단일 API 호출로 처리
  • 비용 상한선 — 월 API 비용 $50 초과 시 알림 + 자동 제한
예상 API 비용: 사용자 100명 × 월 20회 판정 = 2,000회
2,000 × 300 tokens × $0.00025/1K = 월 약 $0.15 (≈ 200원)
