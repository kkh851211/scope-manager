토큰 사용량 & 비용 추정
// 이슈 5건 처리 기준 토큰 분해 System Prompt : ~400 tokens (역할 정의 + 컨텍스트 + 출력 스키마) User Input : ~600 tokens (이슈 5건 × 평균 120토큰) Output (JSON) : ~500 tokens (구조화 결과 5건 + 총합) ───────────────── Total / call : 1,500 tokens // GPT-4o-mini 기준 건당 비용 Input cost : 1,000 tokens × $0.150/1M = $0.00015 (약 0.2원) Output cost : 500 tokens × $0.600/1M = $0.00030 (약 0.4원) Total/call : $0.00045 (약 0.6원) // 월간 비용 시뮬레이션 (유저당 월 평균 3회 호출 가정) 100명 : 300 calls → 약 180원 1,000명: 3,000 calls → 약 1,800원 10,000명: 30,000 calls → 약 18,000원 ← Freemium 수익으로 충분히 커버
오류 처리 & 폴백 전략
JSON 파싱 실패response_format: json_object 강제로 발생 빈도 <1%. 실패 시 자동 1회 재시도, 2회 연속 실패 시 사용자에게 "다시 시도" UI 표시.
타임아웃 (15초)OpenAI 서버 장애 대비. 타임아웃 시 Claude Haiku로 폴백 (백업 키 설정). 실패 메시지는 "AI가 잠시 바빠요, 재시도 해주세요".
스키마 불일치필수 필드(impact_low, impact_high) 누락 시 Zod 스키마 검증 후 400 에러. System Prompt 스키마 예시 보강으로 빈도 감소.
비용 폭주 방지유저당 일 최대 10회 호출 Rate Limit. Supabase RLS로 호출 횟수 카운팅. 초과 시 다음 날 리셋 안내.

기술 리스크 & 대응 방안

발생 가능성 × 임팩트 기준 우선순위 정렬

🔴
R01 — LLM 금액 산출 품질 미달 (H2 가설 실패)
가능성 HIGH임팩트 CRITICAL
리스크 내용
AI 수용률 < 50% — 실무자가 AI 산출 금액을 신뢰하지 못해 대폭 수정하거나 이탈. 핵심 가치 제안 자체가 무력화되는 최우선 위험.
발생 원인
직군별 M/M 단가 편차 미반영, 이슈 유형 분류 오류, 금액 스케일 과장·과소, 산정 근거 불투명.
대응 방안
배포 전: 직군 5종 × 이슈 4유형 = 20케이스 내부 테스트. Few-shot 예시 5건 이상 포함.
배포 후: edit_delta_pct 실시간 모니터링 → 1주 스프린트 프롬프트 개선.
🔴
R02 — OpenAI API 장애 / 가격 인상
가능성 MID임팩트 HIGH
리스크 내용
MVP 핵심 기능이 외부 API에 완전 의존. OpenAI 장애 시 서비스 전면 중단. 가격 인상 시 단위 경제 악화.
발생 원인
단일 LLM 공급자 의존. OpenAI 역사적으로 연간 수회 장애 발생.
대응 방안
멀티 공급자 폴백: OpenAI 실패 시 Claude Haiku 자동 전환 (환경변수로 백업 키 관리).
비용 상한: 월별 LLM 예산 상한 설정, 초과 시 알림.
🟡
R03 — PDF 생성 속도 / 레이아웃 깨짐
가능성 MID임팩트 MID
리스크 내용
클라이언트 사이드 PDF 렌더링 시 폰트 로딩 지연 (3초 초과) 또는 한글 폰트 깨짐. Activation 이벤트 완성 직전 이탈 유발.
발생 원인
@react-pdf의 한글 폰트 번들링 이슈. 모바일 저사양 기기에서 렌더링 지연.
대응 방안
폰트 사전 로딩: Noto Sans KR woff2 서브셋으로 압축 (500KB 이내). 생성 중 스켈레톤 UI + 진행 상태 표시. 3초 초과 시 서버 사이드 렌더링으로 전환 검토.
🟡
R04 — 이벤트 트래킹 누락 (검증 인프라 오염)
가능성 MID임팩트 HIGH
리스크 내용
핵심 이벤트 누락 시 가설 검증 불가 → 전략적 의사결정 오류. 특히 ai_result_edited의 delta_pct 누락 시 H2 보조 가설 측정 불가.
발생 원인
광고 차단기(adblocker)의 Mixpanel 차단, 네트워크 오류 시 이벤트 소실, 코드 배포 시 이벤트 누락.
대응 방안
배포 전 체크리스트: 전 이벤트 수동 발화 확인. 서버 사이드 이벤트(Mixpanel Server API)로 중요 이벤트 이중 수집. DB에도 이벤트 백업 저장.
🟢
R05 — Supabase 무료 플랜 한도 초과
가능성 LOW임팩트 MID
리스크 내용
월 500MB DB, 1GB Storage 한도. 가입자 급증 시 초과 가능성 (약 10,000+ 유저 수준에서 발생 추정).
발생 원인
무료 플랜의 구조적 한계. 증빙 이미지 첨부 유저 급증 시 Storage 먼저 소진.
대응 방안
이미지 압축 (WebP 변환, 1MB 상한) + 30일 후 자동 삭제 정책. Pro 업그레이드 $25/월로 충분히 수용. 좋은 문제 (트래픽 급증 신호).
