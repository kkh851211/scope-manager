Analytics Verification 전략 & 스택

인터뷰 대신 행동 데이터로 가설 검증 — 트래킹 연동 정확성이 MVP의 눈

왜 Analytics Verification이 MVP 배포의 전제 조건인가
🎯 가설 검증 = 이벤트 데이터

인터뷰를 생략하고 MVP를 즉시 배포하는 전략에서, 트래킹 이벤트가 발화되지 않으면 가설 검증 자체가 불가능하다. 트래킹 오류 = 제품이 눈 먼 채로 운영되는 것.

📊 "측정 불가 = 개선 불가"

Activation 율, WAU, AI 정확도 등 린 캔버스 Key Metrics는 모두 이벤트 기반. 이벤트 속성 하나가 누락되면 어느 단계에서 이탈하는지 알 수 없다.

⚡ 배포 전 반드시 검증

배포 후 트래킹 오류 발견 시 이미 수집된 데이터는 복구 불가. W1 데이터가 오염되면 4주 의사결정 타임라인 전체가 무효화된다.

이벤트 페이로드 검증

DevTools Network 탭 + Mixpanel Live View로 이벤트별 속성값 정확성 검증

검증 방법 — DevTools Network 탭
① Network 탭 열기

F12 → Network 탭 → Filter: api.mixpanel.com 입력 → 이벤트 트리거 후 요청 클릭

② Payload 확인

Request URL → Payload 탭 → data 파라미터 Base64 디코딩 → JSON 속성값 육안 검증

③ Live View 대조

Mixpanel → Live View → 해당 이벤트 클릭 → Properties 패널에서 속성값 최종 확인

이상 감지 (Anomaly Alert) 설정
// AI API 오류율 급증 감지 — 5분 롤링 윈도우 const errorWindow = { errors: 0, total: 0, windowStart: Date.now() }; function trackAICall(success, responseTimeMs) { const now = Date.now(); // 5분 윈도우 리셋 if (now - errorWindow.windowStart > 5 * 60 * 1000) { errorWindow.errors = 0; errorWindow.total = 0; errorWindow.windowStart = now; } errorWindow.total++; if (!success) errorWindow.errors++; // 오류율 30% 초과 + 최소 10건 이상 시 알림 const errorRate = errorWindow.errors / errorWindow.total; if (errorWindow.total >= 10 && errorRate > 0.3) { sendSlackAlert('#alerts-critical', `🚨 AI API 오류율 ${(errorRate*100).toFixed(1)}% — 즉시 확인 필요`, '🔴'); } // 응답 시간 이상 감지 (p95 > 12초) if (responseTimeMs > 12000) { sendSlackAlert('#alerts-warn', `⚠️ AI 응답 지연 ${responseTimeMs}ms — LLM 상태 확인`, '🟡'); } }
