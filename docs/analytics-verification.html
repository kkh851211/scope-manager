Analytics Verification — 검증 전략

MVP 행동 데이터가 가설 검증의 유일한 신호. 트래킹이 틀리면 판단도 틀린다.

왜 Analytics Verification인가

고객 인터뷰 대신 행동 데이터로 가설을 검증하는 전략을 선택했기 때문에, 트래킹 데이터의 정확성이 곧 의사결정의 정확성이다.

  • 잘못된 이벤트 = 잘못된 피벗 결정
  • 누락 이벤트 = 보이지 않는 이탈 구간
  • 중복 이벤트 = 과장된 지표, 잘못된 CVR
  • PII 포함 이벤트 = 개인정보보호법 위반
트래킹 검증은 기능 QA와 동등한 우선순위

검증 3계층

Layer 1 · 설치 검증

PostHog SDK 초기화 확인, identify() 호출, 공통 속성 주입

Layer 2 · 이벤트 검증

각 이벤트 발화 여부, 속성 정확성, 중복/누락 없음

Layer 3 · 퍼널 검증

가설별 퍼널 데이터 흐름, Drop-off 구간 식별

PostHog 설치 & 초기화 검증

SDK 설치부터 사용자 식별(identify)까지 단계별 구현 코드 및 검증 방법

Step 1 — 패키지 설치
# Next.js 프로젝트에 PostHog 설치
npm install posthog-js

# 환경변수 설정 (.env.local)
NEXT_PUBLIC_POSTHOG_KEY=phc_YOUR_PROJECT_KEY
NEXT_PUBLIC_POSTHOG_HOST=https://app.posthog.com
⚠ API 키를 .env.local에만 보관. .env 파일은 git에 커밋하지 않는다. .gitignore에 .env.local 포함 확인
Step 2 — SDK 초기화 (Next.js App Router)
// app/providers.tsx
'use client'
import posthog from 'posthog-js'
import { PostHogProvider } from 'posthog-js/react'
import { useEffect } from 'react'

export function PHProvider({ children }) {
  useEffect(() => {
    posthog.init(process.env.NEXT_PUBLIC_POSTHOG_KEY, {
      api_host: process.env.NEXT_PUBLIC_POSTHOG_HOST,
      capture_pageview: false, // 수동 제어
      capture_pageleave: true,
      persistence: 'localStorage',
      autocapture: false, // 의도치 않은 수집 차단
    })
  }, [])
  return <PostHogProvider client={posthog}>{children}</PostHogProvider>
}
Step 3 — 사용자 식별 (identify)
// 로그인 완료 후 호출
import { usePostHog } from 'posthog-js/react'

const posthog = usePostHog()

// 로그인 성공 시
posthog.identify(
  hashedUserId, // UUID hash (이메일 원문 X)
  {
    agency_size: profile.agency_size,
    plan: 'beta_free',
    created_at: profile.created_at,
    onboarded: profile.onboarded,
  }
)

// 로그아웃 시 반드시 호출
posthog.reset()
Step 4 — 공통 속성 유틸리티
// lib/analytics.ts
export const commonProps = () => ({
  platform: 'web',
  app_version: process.env.NEXT_PUBLIC_APP_VERSION,
  timestamp: new Date().toISOString(),
})

// 사용 예시
export const track = (event, props = {}) => {
  posthog.capture(event, {
    ...commonProps(),
    ...props,
  })
}

// 실제 사용
track('judgment_completed', {
  verdict: 'OUT',
  confidence: 0.92,
  is_first_judgment: true,
})

설치 검증 체크리스트

🔧 DevTools 검증
  • Network 탭 → app.posthog.com/capture 요청 확인
  • 요청 status 200 확인
  • payload에 distinct_id 포함 확인
  • payload에 이메일 원문 미포함 확인
  • Console 탭에 SDK 초기화 오류 없음
📊 PostHog 대시보드 검증
  • People 탭에 테스트 유저 표시
  • Properties에 agency_size, plan 반영
  • Session Recordings 활성화 확인
  • Feature Flags 연결 테스트
🔒 보안 검증
  • 이메일 원문이 이벤트 속성에 없음
  • 비밀번호가 어떤 이벤트에도 없음
  • 계약서 원문이 이벤트에 없음
  • user_id는 UUID hash만 사용
  • autocapture=false 유지

PostHog Toolbar 활성화 방법

PostHog 대시보드 → Settings → Toolbar 메뉴
사이트 URL 등록 (예: https://staging.scopemanager.io)
Authorize Toolbar 클릭 → 스테이징 사이트 이동
화면 하단 PostHog 아이콘 → Inspector 모드 활성화
버튼 클릭 시 발화되는 이벤트 실시간 확인
✅ 코드 수정 없이 화면에서 직접 이벤트 발화를 시각적으로 확인 가능

페이지뷰 수동 제어 (Next.js Router)

// app/layout.tsx — 라우트 변경 감지
'use client'
import { usePathname } from 'next/navigation'
import { usePostHog } from 'posthog-js/react'
import { useEffect } from 'react'

export function PageViewTracker() {
  const pathname = usePathname()
  const posthog = usePostHog()
  useEffect(() => {
    posthog.capture('$pageview', {
      $current_url: window.location.href,
    })
  }, [pathname])
  return null
}

// Chrome DevTools Console에 붙여넣기 → 모든 PostHog 요청 모니터링
const origFetch = window.fetch
window.fetch = async function(...args) {
  const url = args[0]
  if (typeof url === 'string' && url.includes('posthog')) {
    const body = args[1]?.body
    if (body) {
      try {
        const parsed = JSON.parse(decodeURIComponent(body).replace('data=', ''))
        console.group(`📊 PostHog: ${parsed.batch?.[0]?.event}`)
        console.log('Properties:', parsed.batch?.[0]?.properties)
        console.groupEnd()
      } catch(e) {}
    }
  }
  return origFetch(...args)
}

이벤트 검증 규칙 — 콘솔에서 자동 감지

🔴 FAIL 자동 감지
  • PII 키 포함 (email, name, phone)
  • 계약서 원문 1000자+ 포함
  • 동일 이벤트 2초 내 중복 발화
  • aha_moment_reached 2회 이상
  • 이벤트명에 대문자 포함
⚠️ WARNING 자동 감지
  • timestamp 속성 누락
  • user_id 속성 누락
  • 응답 시간 5000ms 초과
  • confidence 값이 0~1 범위 외
  • verdict가 IN/OUT/AMBIGUOUS 외 값
✅ PASS 기준
  • 이벤트명 snake_case 준수
  • 공통 속성 전체 포함
  • PII 키 없음
  • 중복 발화 없음
  • PostHog /capture 200 응답
