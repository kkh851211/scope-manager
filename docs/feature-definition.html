기능 범위 정의

MVP에 담는 것과 담지 않는 것 — 린 캔버스 v6.0 Scope Creep 차단 원칙

✅ In-Scope — MVP에 포함
소급 텍스트 입력 — 과거 이슈 최대 5건 자유 기술
AI 구조화 & 금액 환산 — 이슈 → 재무 임팩트 초안 산출
Human-in-the-loop 수정 — 금액·설명 편집, 증빙 캡처 첨부
피치덱 PDF 원클릭 생성 — 연봉 협상용 1페이지 리포트
기본 온보딩 — 직군·역할·재직기간 입력 (AI 컨텍스트용)
이벤트 트래킹 — 가입·입력·완성·다운로드 이벤트 수집
랜딩 페이지 — 문제 공감 + 가입 유도 + UTM 파라미터
❌ Out-of-Scope — MVP에서 제외
이메일·메신저 자동 파싱 — 연동 복잡도 대비 검증 가치 낮음
B2B 경영진 대시보드 — WTP 검증 후 개발 (Phase 2)
팀/조직 단위 기능 — 개인 검증 완료 전 불필요
급여·KPI 시스템 연동 — 대기업 영역, Out-of-Scope 선언
소셜 공유·커뮤니티 — 핵심 루프 검증 후 추가
모바일 앱 — 웹 검증 우선, 앱은 Phase 3 이후
다국어·다통화 지원 — 국내 SME SOM 집중 단계
기능 우선순위 판단 기준 — MoSCoW
Must Have — 없으면 MVP 아님
텍스트 입력 → AI 환산 → 피치덱 생성. 이 3단계 중 하나라도 빠지면 핵심 가설 검증 불가.
Should Have — 품질 확보에 필요
Human-in-the-loop 수정, 증빙 첨부. 없어도 동작하지만 AI 수용률 지표에 직접 영향.
Could Have — 있으면 좋지만 나중에
이벤트 트래킹 고도화, 리마인더 이메일, 공유 버튼. Activation 이후 단계에서 추가.
Won't Have — 이번 MVP에서 하지 않음
자동 파싱, B2B 대시보드, 앱, 팀 기능. Out-of-Scope 항목과 동일. 명시적으로 제외 선언.

MVP 기능 상세 정의

F01~F06 — 각 기능의 목적·동작 방식·인수 조건

F01
랜딩 페이지 & 온보딩
MustMVP
목적 & 가설 연결
문제 공감 메시지로 H1 CVR ≥ 15% 달성. 가입 시 직군·역할·재직기간 수집 → AI 프롬프트 컨텍스트로 활용.
핵심 동작
① 헤드라인 A/B 테스트 ("연봉 협상 무기" vs "숨은 성과 증명")
② 소셜 로그인(Google) 또는 이메일 가입
③ 온보딩 3문항: 직군 / 회사 규모 / 현재 상황(협상 준비 여부)
인수 조건 (AC)
• UTM 파라미터 자동 저장
• 가입 완료 시 signup_complete 이벤트 발화
• 온보딩 3문항 모두 필수 (스킵 불가)
• 모바일 반응형
예외 처리
소셜 로그인 실패 시 이메일 가입으로 자동 폴백
온보딩 이탈 시 미완성 상태 저장 후 재진입 시 이어서 진행
F02
소급 이슈 텍스트 입력
MustMVP 핵심
목적 & 가설 연결
실무자가 과거 이슈 1~5건을 자유 텍스트로 기술. 입력 행동 자체가 H1 페인 강도 측정. 이슈 유형(예방/수습/역할중첩)은 AI가 자동 분류.
핵심 동작
① 이슈 1건당 입력 필드: 상황 설명(필수) + 예상 임팩트(선택)
② 입력 중 실시간 힌트 문구 제공 ("언제, 무슨 문제를, 어떻게 막았나요?")
③ 1건 입력 후 "추가하기" 버튼으로 최대 5건까지 확장
인수 조건 (AC)
• 최소 1건 입력 시 AI 분석 버튼 활성화
• 각 필드 최소 50자 / 최대 500자
• issue_input_start, issue_input_3_complete 이벤트 발화
• 임시저장 자동 (30초 간격)
UX 세부 사항
입력창 포커스 시 이슈 유형 예시 툴팁 표시 (사전 방어 / 위기 수습 / 역할 중첩)
글자 수 카운터 표시 (50자 미만 시 경고 색상)
이슈 삭제·순서 변경 드래그 앤 드롭 (Could Have — Phase 2)
F03
AI 구조화 & 재무 금액 환산
MustMVP 핵심
목적 & 가설 연결
자유 텍스트를 "이슈 유형 / 방어 임팩트 / 예상 절감액" 3요소로 구조화. H2 핵심 기술 가설 — AI 산출값 수용률 ≥ 50% 검증.
핵심 동작
① 온보딩 컨텍스트(직군·규모) + 이슈 텍스트 → LLM 호출
② 출력: 이슈 요약 / 유형 분류 / 방어 금액 범위(하한~상한) / 산정 근거
③ 처리 중 로딩 애니메이션 + 예상 소요시간 표시 (5~10초)
인수 조건 (AC)
• LLM 응답 실패 시 재시도 버튼 + 오류 메시지
• 응답 타임아웃 15초 설정
• ai_result_shown 이벤트 발화
• 금액 단위: 만원 (소수점 없음)
산정 근거 투명화 (신뢰도 핵심)
금액 하단에 산정 방식 1줄 요약 노출 ("M/M 비용 × 예상 지연 시간 기준")
수용률 < 50% 시 — 근거 상세 펼치기 기능 추가 (Phase 1.5 스프린트)
F04
Human-in-the-loop 수정 & 증빙 첨부
ShouldMVP 포함
목적 & 가설 연결
AI 산출값에 대한 실무자의 최종 통제권 보장. 금액 수정 폭(±%) 측정으로 H2 보조 가설 검증. 캡처 첨부로 피치덱 신뢰도 향상.
핵심 동작
① AI 결과 각 항목 인라인 편집 (이슈 요약·금액·근거)
② 금액 슬라이더 또는 직접 입력 (AI 값 ±50% 범위 초기 설정)
③ 이슈별 증빙 이미지 첨부 (선택, 최대 3장)
인수 조건 (AC)
• 수정 전·후 금액 차이 edit_delta_pct 로깅
• 이미지 파일 5MB 이하, JPG/PNG/WebP
• "AI 권장값으로 복원" 버튼 제공
• ai_result_accepted / ai_result_edited 이벤트 발화
F05
피치덱 PDF 원클릭 생성
MustNorth Star
목적 & 가설 연결
핵심 Activation 이벤트. 완성된 피치덱이 "연봉 협상 무기"라는 확신을 주는지 — PDF 완성률 ≥ 40% + 다운로드율 측정.
핵심 동작
① 1페이지 A4 PDF: 이름·직군·기간 / 이슈 요약 카드 / 총 방어 금액 / 기여 유형 분포
② 브랜드 커버 디자인 (MyROI 로고 + 날짜)
③ "다운로드" + "링크 복사" 2가지 공유 방식
인수 조건 (AC)
• pitchdeck_generated, pitchdeck_downloaded 이벤트
• PDF 생성 3초 이내
• 링크 공유 시 뷰어 전용 웹 버전 제공
• 피치덱 하단 B2B Waitlist 티저 문구 삽입
B2B 연결 포인트 (WTP 검증 내장)
PDF 하단 고정 문구: "이 데이터를 팀 전체로 확장하고 싶으신 대표님 →" Waitlist 링크
링크 클릭 시 buyer_landing_visit 이벤트 발화 → H5 검증 시작
F06
이벤트 트래킹 & 분석 인프라
MustMVP 전제
목적
모든 가설 검증의 측정 인프라. 이벤트가 누락되면 어떤 가설도 검증 불가. MVP 배포 전 반드시 완료.
트래킹 항목
landing_visit (UTM) / signup_complete / issue_input_start / issue_input_3_complete / ai_result_shown / ai_result_accepted / ai_result_edited (delta_pct) / pitchdeck_generated / pitchdeck_downloaded / d7_return / d30_return
인수 조건 (AC)
• 이벤트 누락률 0% (배포 전 QA 체크리스트)
• 채널별 UTM 자동 태깅
• 코호트 분석을 위한 user_id + first_visit_date 저장
• 개인정보: 금액·텍스트는 서버 로그에서 마스킹

AI 프롬프트 명세

H2 기술 가설의 핵심 — 텍스트 → 재무 임팩트 변환 프롬프트 설계

프롬프트 설계 원칙
원칙 1 컨텍스트 최대화
온보딩에서 수집한 직군·회사 규모·재직기간을 System Prompt에 주입. 동일한 이슈도 직군에 따라 금액 스케일이 다르다 (PM의 일정 지연 방어 vs 개발자의 서버 장애 방어).
직군 컨텍스트 주입M/M 단가 추정규모별 보정

System Prompt 구조 (의사코드)
## 역할 당신은 중소기업 실무자의 비공식 기여를 재무적 임팩트로 변환하는 전문가입니다. ## 컨텍스트 (온보딩 데이터 주입) - 직군: {job_role} ← e.g. "PM / 기획자" - 회사 규모: {company_size} ← e.g. "20~50인" - 재직기간: {tenure} ← e.g. "3년" - 추정 M/M 단가: {estimated_mm_cost} ← 직군·규모 기반 자동 계산 ## 이슈 유형 분류 기준 1. 사전예방 (Proactive Prevention): 사고·지연을 막은 사전 행동 2. 위기수습 (Reactive Resolution): 발생한 문제를 해결한 사후 행동 3. 역할중첩 (Role Overlap): 본업 외 조직 공백을 메운 헌신 ## 출력 형식 (JSON 필수, 마크다운 금지) { "issues": [ { "summary": "이슈 1줄 요약", "type": "사전예방 | 위기수습 | 역할중첩", "impact_low": 숫자(만원), "impact_high": 숫자(만원), "calculation_basis": "산정 근거 1~2문장", "confidence": "high | medium | low" } ], "total_low": 숫자, "total_high": 숫자, "summary_sentence": "전체 기여 1줄 요약" } ## 제약 조건 - 금액은 보수적으로 산정 (과장 금지, 신뢰 우선) - confidence가 low인 경우 이유를 calculation_basis에 명시 - 정보 부족 시 추가 질문 없이 low confidence로 산정
